Project 1 --- Basic Principles and Models¶

We're going to work with the California housing dataset. This dataset was derived from the 1990 U.S. census, using one row per census block group. A block group is the smallest geographical unit for which the U.S. Census Bureau publishes sample data (a block group typically has a population of 600 to 3,000 people). The target variable is the median house value for California districts, expressed in hundreds of thousands of dollars ($100,000).

  • Standardization of the two features reduces the scale of both x-axis and y-axis. After standardization, the data have a common scale with a mean of 0 and a standard deviation of 1, effectively removing the original feature units.
  • The scatter plots before and after standardization look similar because standardization only changes the scale but keeps the data pattern to be the same. It applies the same operation on both x-value and y-value. Therefore, the relationships between the features preserve while only removing the original units.
  • Standardization creates uniformity and small-scale comparison, making it easier to identify certain values and compare them across features. It also improves the performance and training stability without altering the data patterns.

In this part, we aim to investigate the importance of each feature on the final classification accuracy. If we want to try every possible combination of features, we would have to test $2^F$ different cases, where F is the number of features, and in each case, we have to do a hyperparameter search (finding K, in KNN using cross-validation). That will take days!.

To find more important features we will use a decision tree. based on a decision tree we can compute feature importance that is a metric for our feature selection (code is provided below).

After we identified and removed the least important feature and evaluated a new KNN model on the new set of features, if the stop conditions (see step 7 below) are not met, we need to repeat the process and remove another feature.

from sklearn import tree

# find the best cv score and k value using KNN
def Find_best_k(X_train, y_train, k_range=(1, 101, 5)):
    best_cv = 0
    best_k = 1
    for k in range(*k_range):
        knn = KNeighborsClassifier(n_neighbors=k)
        cv_score = cross_val_score(knn, X_train, y_train, cv=5).mean()
        if cv_score > best_cv:
            best_cv = cv_score
            best_k = k
    return best_cv, best_k

def Feature_selector (X_train , y_train , tr=0.95):
    n_features = X_train.shape[1]  # number of remaining features
    features = list(X_train.columns)  # list of remaining feature names
    removed_features, cv_scores, best_ks = [], [], []

    dt = tree.DecisionTreeClassifier()
    dt.fit(X_train, y_train)

    full_cv_score = cross_val_score(KNeighborsClassifier(), X_train, y_train, cv=5).mean()
    best_cv = full_cv_score

    # feature selection process
    while len(features) > 1 and best_cv >= tr * full_cv_score:
        dt.fit(X_train[features], y_train) # fit decision tree
        importance = dt.feature_importances_ # get feature importances

        # remove the least important feature
        least_important = features[importance.argmin()]
        removed_features.append(least_important)
        features.remove(least_important)

        # find the best cv and k
        best_cv, best_k = Find_best_k(X_train[features], y_train)

        # store the best results for the current number of features
        cv_scores.append(best_cv)
        best_ks.append(best_k)

    return removed_features, cv_scores, best_ks

Part 4: Standardization¶

Standardizing the data usually means scaling our data to have a mean of zero and a standard deviation of one.

Note: When we standardize a dataset, do we care if the data points are in our training set or test set? Yes! The training set is available for us to train a model - we can use it however we want. The test set, however, represents a subset of data that is not available for us during training. For example, the test set can represent the data that someone who bought our model would use to see how the model performs (which they are not willing to share with us). Therefore, we cannot compute the mean or standard deviation of the whole dataset to standardize it - we can only calculate the mean and standard deviation of the training set. However, when we sell a model to someone, we can say what our scalers (mean and standard deviation of our training set) was. They can scale their data (test set) with our training set's mean and standard deviation. Of course, there is no guarantee that the test set would have a mean of zero and a standard deviation of one, but it should work fine.

To summarize: We fit the StandardScaler only on the training set. We transform both training and test sets with that scaler.
